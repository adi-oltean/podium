# podium_nav_app — a cFS relative-navigation app on verified kernels

Generated by `podium.emit.cfsapp`. This is a Core Flight System (cFS)
application whose GNC arithmetic is **entirely emitted, EVA-proven,
CompCert-checked C**: the measurement handler calls
`podium_ekf_update_sequential` then `podium_ekf_predict` over a
maintained 6-state estimate and 6x6 covariance.

## Files

- `podium_nav_app.c/.h` — the app, in genuine cFS idiom.
- `cfe_shim.c/.h` — a minimal, portable stand-in for the cFE SB/ES/EVS
  API so the app compiles and runs standalone (no OSAL/full cFE). It is
  NOT cFE; swap it for the real `cfe.h` in a mission build unchanged.
- `podium_kernels.c` — the emitted flight kernels (copied in by the
  generator).

## Build & run standalone

    gcc -std=c99 -O2 -ffp-contract=off \
        podium_nav_app.c cfe_shim.c podium_kernels.c -lm -o nav_app
    ./nav_app < measurements.txt   # each line: 3 hex-float positions

Every transmitted STATE message prints as 6 hex floats — the same `%a`
transport the golden-vector tests use, so the output is directly
comparable to the Python EKF.

## Mapping to a real cFS mission

| this example                     | real cFS                          |
| -------------------------------- | --------------------------------- |
| `PODIUM_NAV_AppMain`             | app entry in the ES app table     |
| `cfe_shim.h`                     | `cfe.h` (SB/ES/EVS/MSG)           |
| `CFE_SB_CreatePipe/Subscribe`    | identical                         |
| `CFE_SB_ReceiveBuffer` loop      | identical (`CFE_ES_RunLoop`)      |
| `PODIUM_NAV_*_MID`               | mission-assigned message IDs      |
| `cfe_shim.c` line transport      | the real Software Bus             |

## Mapping to F´ (F Prime)

| this example                     | F´                                |
| -------------------------------- | --------------------------------- |
| `PODIUM_NAV_App_t`               | a `Component`                     |
| MEAS subscription                | an input (async) `port`           |
| `PODIUM_NAV_OnMeas` handler      | the port handler                  |
| STATE transmit                   | an output `port` / telemetry chan |
| init constants (phi, Q, r_var)   | `parameters`                      |
| (add) reset/init command         | a `command`                       |

The verified kernels are the same in both; only the bus glue differs.
