Podium flies the SpaceX ISS docking simulator

Podium's pulsed docking-control laws, running as an autopilot inside iss-sim.spacex.com.

What this is

SpaceX's public simulator models Crew Dragon's manual docking mode: every control input is one pulse — a fixed velocity or attitude-rate increment. That is exactly the minimum-impulse-bit actuation model of podium.control.docking. The script below is a line-for-line JavaScript port of those laws (Python is the source of truth; constants are identical). It reads the sim's own HUD telemetry from the page and clicks the sim's own control buttons — Podium GNC in the loop, SpaceX's vehicle model.

closing-rate profilev = clamp(range/60, 0.16, 3.0) m/s
lateral / attitude lawsaturated proportional, gain-scheduled near the port
pulse allocatorhalf-pulse deadband — no thruster chatter
docking gates (the sim's)<0.2 m lateral, <0.2° per axis, <0.2 m/s at contact

Run it

  1. Open iss-sim.spacex.com and press BEGIN.
  2. Open the browser devtools console (F12 or Ctrl/Cmd+Shift+J).
  3. Chrome will ask you to type allow pasting first — do that.
  4. Copy the script below, paste into the console, press Enter.
  5. Watch. Telemetry logs every 5 s; podiumAP.stop() disengages.
// Podium autopilot for iss-sim.spacex.com
// Port of podium.control.docking (github.com/adi-oltean/podium) — MIT.
// Reads the sim's HUD telemetry from the DOM, actuates via the sim's own
// control buttons. Educational demo; not affiliated with SpaceX.
(() => {
if (window.podiumAP) { console.warn("[podium] already engaged"); return; }

// ---- Podium control laws (parity with podium/control/docking.py) ----
const CONTACT_RATE = 0.16, FAR_RATE = 3.0, APPROACH_TAU = 60.0;
const LATERAL_V_MAX = 0.5, LATERAL_TAU = 25.0, LATERAL_TAU_NEAR = 4.0, LATERAL_NEAR_ZONE = 2.0;
const ANGLE_RATE_MAX = 0.5, ANGLE_TAU = 10.0, ANGLE_TAU_NEAR = 2.0, ANGLE_NEAR_ZONE = 2.0;
const clamp = (v, lo, hi) => Math.min(hi, Math.max(lo, v));
const approachRate = r => clamp(r / APPROACH_TAU, CONTACT_RATE, FAR_RATE);
const axisRateCmd = (p, vmax, tau) => clamp(-p / tau, -vmax, vmax);
const lateralRateCmd = p =>
  axisRateCmd(p, LATERAL_V_MAX, Math.abs(p) > LATERAL_NEAR_ZONE ? LATERAL_TAU : LATERAL_TAU_NEAR);
const angleRateCmd = e =>
  axisRateCmd(e, ANGLE_RATE_MAX, Math.abs(e) > ANGLE_NEAR_ZONE ? ANGLE_TAU : ANGLE_TAU_NEAR);
const pulsesNeeded = (vc, ve, dv) =>
  clamp(Math.trunc((vc - ve) / dv + (vc > ve ? 0.5 : -0.5)), -25, 25);

// ---- sim interface ----
const num = sel => {
  const el = document.querySelector(sel);
  const v = parseFloat((el?.textContent || "").replace(/[^\d.eE+-]/g, ""));
  return Number.isFinite(v) ? v : 0;
};
const telemetry = () => ({
  x: num("#x-range .distance"), y: num("#y-range .distance"), z: num("#z-range .distance"),
  range: num("#range .rate"), rate: num("#rate .rate"),
  pitch: num("#pitch .error"), pitchRate: num("#pitch .rate"),
  yaw: num("#yaw .error"), yawRate: num("#yaw .rate"),
  roll: num("#roll .error"), rollRate: num("#roll .rate"),
});
const click = id => {
  const el = document.getElementById(id);
  if (!el) return;
  for (const t of ["pointerdown", "mousedown", "pointerup", "mouseup", "click"])
    el.dispatchEvent(new MouseEvent(t, { bubbles: true, cancelable: true, view: window }));
};
// n>0 clicks pos button, n<0 clicks neg, spaced 70 ms
const fire = (pos, neg, n) => {
  const id = n > 0 ? pos : neg;
  for (let i = 0; i < Math.abs(n); i++) setTimeout(() => click(id), 70 * i);
};

// Pulse sizes (sim fine mode). DV_ROT re-estimated from HUD rate deltas;
// DV_TRANS re-estimated from velocity-slope deltas after our own pulses.
let DV_ROT = 0.1, DV_TRANS = 0.05;

// ---- rotation axes: HUD gives error AND rate; sign learned in flight ----
// map: [error sel, +rate button, -rate button, learned sign s (errDot = s*rate)]
const ROT = {
  pitch: { pos: "pitch-up-button", neg: "pitch-down-button", s: 1, held: 0 },
  yaw:   { pos: "yaw-left-button",  neg: "yaw-right-button",  s: 1, held: 0 },
  roll:  { pos: "roll-left-button", neg: "roll-right-button", s: 1, held: 0 },
};
// ---- translation lateral axes: velocity estimated by finite difference ----
const LAT = {
  y: { pos: "translate-left-button", neg: "translate-right-button", s: 1, v: 0, cum: 0 },
  z: { pos: "translate-up-button",   neg: "translate-down-button",  s: 1, v: 0, cum: 0 },
};

const hist = [];               // telemetry ring buffer, ~4 Hz
let engaged = true, t0 = Date.now(), lastLog = 0;

const sample = setInterval(() => {
  if (!engaged) return;
  const T = telemetry();
  T.t = (Date.now() - t0) / 1000;
  hist.push(T);
  if (hist.length > 8) hist.shift();
}, 250);

const slope = key => {            // least-squares slope over the buffer [unit/s]
  if (hist.length < 5) return 0;
  const n = hist.length, mt = hist.reduce((a, h) => a + h.t, 0) / n,
    mv = hist.reduce((a, h) => a + h[key], 0) / n;
  let num_ = 0, den = 0;
  for (const h of hist) { num_ += (h.t - mt) * (h[key] - mv); den += (h.t - mt) ** 2; }
  return den > 0 ? num_ / den : 0;
};

const ctl = setInterval(() => {
  if (!engaged) return;
  const T = telemetry();
  if (!T.range && !T.x) return;                    // not started yet
  const range = T.range || Math.hypot(T.x, T.y, T.z);

  // --- attitude: track angleRateCmd on each axis using HUD rates ---
  for (const [k, ax] of Object.entries(ROT)) {
    const err = T[k], rate = T[k + "Rate"];
    // in-flight sign check: if error keeps growing against the command, flip
    if (Math.abs(err) > 0.5 && Math.sign(rate) === Math.sign(err) * ax.s && ++ax.held > 6) {
      ax.s = -ax.s; ax.held = 0; console.log(`[podium] flipped ${k} sign`);
    } else if (ax.held > 0 && Math.sign(rate) !== Math.sign(err) * ax.s) ax.held = 0;
    const n = pulsesNeeded(ax.s * angleRateCmd(err), rate, DV_ROT);
    fire(ax.pos, ax.neg, n);
  }

  // --- lateral translation: estimated velocity, learned sign ---
  for (const [k, ax] of Object.entries(LAT)) {
    ax.v = slope(k);
    // sign check: after >=3 net pulses, velocity must have moved with them
    if (Math.abs(ax.cum) >= 3) {
      if (Math.sign(ax.v - (ax.v0 ?? 0)) === -Math.sign(ax.cum) &&
          Math.abs(ax.v - (ax.v0 ?? 0)) > 0.03) {
        ax.s = -ax.s; console.log(`[podium] flipped ${k} sign`);
      }
      ax.cum = 0; ax.v0 = ax.v;
    }
    const n = pulsesNeeded(ax.s * lateralRateCmd(T[k]), ax.v * ax.s, DV_TRANS);
    fire(ax.pos, ax.neg, n);
    ax.cum += n;
  }

  // --- axial: closing-rate profile; HUD "rate" is negative when closing ---
  const closing = -T.rate;                          // m/s toward the port
  const n = pulsesNeeded(approachRate(range), closing, DV_TRANS);
  fire("translate-forward-button", "translate-backward-button", n);

  // --- telemetry log + finish detection ---
  if (T.t - lastLog > 5) {
    lastLog = T.t;
    console.log(`[podium] t=${T.t.toFixed(0)}s range=${range.toFixed(1)}m ` +
      `closing=${closing.toFixed(2)}m/s lat=(${T.y.toFixed(1)},${T.z.toFixed(1)})m ` +
      `pyr=(${T.pitch.toFixed(1)},${T.yaw.toFixed(1)},${T.roll.toFixed(1)})deg`);
  }
  const done = document.querySelector("#success, #fail");
  if (done && getComputedStyle(done).display !== "none" && range < 1) stop("sim ended");
}, 1000);

function stop(why) {
  engaged = false; clearInterval(ctl); clearInterval(sample);
  console.log(`[podium] disengaged${why ? " — " + why : ""}`);
}
window.podiumAP = { stop: () => stop("manual"), telemetry, hist };
console.log("[podium] autopilot engaged — podiumAP.stop() to disengage");
})();

How it works

Three laws, straight from podium/control/docking.py:

1. Closing-rate profile. approach_rate(range) commands an exponential-decay glideslope in range — 3 m/s far out, ramping down to a firm 0.16 m/s at contact (inside the sim's 0.2 m/s gate).

2. Gain-scheduled lateral/attitude nulling. A saturated proportional law with a shorter time constant near the port. The scheduling is not cosmetic: with pulsed actuation, a proportional law stalls once its command drops below half a pulse, leaving a standing offset of tau · Δv/2. With the far-field gains alone that stall offset is 0.6 m / 0.25° — outside the docking gates. The near-zone gains put it at 0.1 m / 0.05°. (Podium's click-dynamics unit tests caught exactly this failure mode.)

3. Half-pulse deadband allocator. pulses_needed() converts a rate error into an integer click count, rounding toward zero inside half a pulse — the vehicle never chatters around its setpoint.

The browser side adds what a real GNC stack would call navigation and FDIR: lateral velocities are estimated by least-squares slope over a 2 s telemetry window (the sim HUD shows no lateral rates), and control-polarity is learned in flight — if an axis responds opposite to its pulses, the sign map flips once and the controller carries on.

Caveats

Educational demo, not affiliated with SpaceX. The script only reads the page's displayed telemetry and clicks its visible buttons — nothing else. If SpaceX changes the page structure, the DOM selectors may need updating. Leave the sim's precision toggles at their defaults; the pulse-size constants assume fine mode and adapt slowly if actual increments differ.
