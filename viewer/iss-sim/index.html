Podium flies the SpaceX ISS docking simulator

Podium's pulsed docking-control laws, running as an autopilot inside iss-sim.spacex.com.

What this is

SpaceX's public simulator models Crew Dragon's manual docking mode: every control input is one pulse — a fixed velocity or attitude-rate increment. That is exactly the minimum-impulse-bit actuation model of podium.control.docking. The script below is a line-for-line JavaScript port of those laws (Python is the source of truth; constants are identical). It reads the sim's own HUD telemetry from the page and clicks the sim's own control buttons — Podium GNC in the loop, SpaceX's vehicle model.

closing-rate profilev = clamp(range/60, 0.16, 3.0) m/s
lateral / attitude lawsaturated proportional, gain-scheduled near the port
pulse allocatorhalf-pulse deadband — no thruster chatter
docking gates (the sim's)<0.2 m lateral, <0.2° per axis, <0.2 m/s at contact

Run it

  1. Open iss-sim.spacex.com and press BEGIN.
  2. Open the browser devtools console (F12 or Ctrl/Cmd+Shift+J).
  3. Chrome will ask you to type allow pasting first — do that.
  4. Copy the script below, paste into the console, press Enter.
  5. Watch. Telemetry logs every 5 s; podiumAP.stop() disengages.
// Podium autopilot for iss-sim.spacex.com
// Port of podium.control.docking (github.com/adi-oltean/podium) — MIT.
// Plant model decoded from the sim source + Playwright sysid (repo:
// tools/ui/sysid*_issim.py):
//   - translation state (motionVector) is meters PER FRAME, applied along
//     capsule BODY axes (displayed x/y/z = body -z/+x/+y); each click adds
//     exactly translationPulseSize (0.001) per frame. The autopilot counts
//     requestAnimationFrame to convert its SI control laws to frame units.
//   - attitude: clicks step a target register; angular velocity is
//     proportional to MINUS that register (the HUD rate is the click
//     counter, sign-inverted vs the true drift). Gain measured at engage,
//     re-estimated in flight.
//   - success gate (from source): raycast contact with each |motionVector|
//     component <= 0.004 and each angle <= 0.2 deg. The contact raycast
//     reaches only 0.1 m at terminal range, so the last meters are flown at
//     0.003 per frame — fast approaches tunnel through the port untouched.
// Actuation calls the sim's own control functions (same as its buttons).
// Educational demo; not affiliated with SpaceX.
(() => {
if (window.podiumAP) { console.warn("[podium] already engaged"); return; }

// ---- Podium control laws, SI units (parity with podium/control/docking.py)
const ANGLE_RATE_MAX = 0.5, ANGLE_TAU = 10.0, ANGLE_TAU_NEAR = 2.0, ANGLE_NEAR_ZONE = 2.0;
const clamp = (v, lo, hi) => Math.min(hi, Math.max(lo, v));
const axisRateCmd = (p, vmax, tau) => clamp(-p / tau, -vmax, vmax);
const angleRateCmd = e =>
  axisRateCmd(e, ANGLE_RATE_MAX, Math.abs(e) > ANGLE_NEAR_ZONE ? ANGLE_TAU : ANGLE_TAU_NEAR);
const latVMax = x => clamp(0.03 + 0.03 * x, 0.05, 0.4);          // m/s, distance-scheduled
const lateralRateCmd = (p, x) => axisRateCmd(p, latVMax(x), Math.abs(p) > 2 ? 25.0 : 8.0);
const approachRateSI = r => Math.min(r / 50.0, 3.0);             // m/s profile (floor in mv below)
const pulsesNeeded = (vc, ve, dv) =>
  clamp(Math.trunc((vc - ve) / dv + (vc > ve ? 0.5 : -0.5)), -40, 40);

// frame-unit floors/caps from the sim's own gates. The cruise ceiling is
// dominated by the SI profile (3 m/s) on fast machines and by the absolute
// frame cap (anti-tunneling) on slow ones.
const MV_CONTACT = 0.003;   // per-frame closing at contact (< 0.004 gate)
const MV_FAR = 0.3;         // per-frame cruise ceiling

// ---- sim interface ----
const num = sel => {
  const el = document.querySelector(sel);
  const v = parseFloat((el?.textContent || "").replace(/[^\d.eE+-]/g, ""));
  return Number.isFinite(v) ? v : 0;
};
const nav = () => {
  const qi = camera.quaternion.clone().conjugate();
  const vb = motionVector.clone().applyQuaternion(qi);   // body frame, m/FRAME
  return {
    x: num("#x-range .distance"), y: num("#y-range .distance"), z: num("#z-range .distance"),
    pitch: num("#pitch .error"), yaw: num("#yaw .error"), roll: num("#roll .error"),
    rr: [rateRotationX, rateRotationY, rateRotationZ],   // click counters
    mvClose: -vb.z, mvy: vb.x, mvz: vb.y,
  };
};
const POS = [() => pitchDown(), () => yawRight(), () => rollRight()];  // err NEGATIVE
const NEG = [() => pitchUp(), () => yawLeft(), () => rollLeft()];
const ERRK = ["pitch", "yaw", "roll"];
const fire = (posFn, negFn, n) => {
  const fn = n > 0 ? posFn : negFn;
  for (let i = 0; i < Math.abs(n); i++) setTimeout(fn, 22 * i);
};
const dvT = () => translationPulseSize;   // m/frame per click (keep default 0.001!)

// frame-rate counter: bridges SI laws to the sim's per-frame world
let frames = 0, fps = 30;
(function cnt() { frames++; requestAnimationFrame(cnt); })();

// ---- engage: wait out the intro, measure the attitude plant gain ----
let phase = "waiting", engaged = true, t0 = Date.now(), lastLog = 0;
let qPrev = null, stableFor = 0, calT = 0, calErr0 = 0, gAbs = 0.05;
let prev = null, prevT = 0;
const camQ = () => [camera.quaternion.x, camera.quaternion.y, camera.quaternion.z, camera.quaternion.w];
const CAL_N = 5;

const ctl = setInterval(() => {
  if (!engaged) return;
  const t = (Date.now() - t0) / 1000;
  const dt = prevT ? t - prevT : 1;
  fps = clamp(frames / dt, 5, 240); frames = 0;

  if (phase === "waiting") {
    const q = camQ();
    const moved = qPrev ? Math.hypot(...q.map((v, i) => v - qPrev[i])) : 1;
    qPrev = q; prevT = t;
    stableFor = moved < 1e-4 ? stableFor + 1 : 0;
    if (stableFor >= 2 && num("#range .rate") > 0 && isIssLoaded) {
      if (translationPulseSize > 0.004) toggleTranslation();  // want fine 0.001 clicks
      calErr0 = nav().pitch;
      fire(POS[0], NEG[0], CAL_N);
      calT = t; phase = "calibrating";
      console.log("[podium] intro settled, ISS mesh loaded — measuring plant gain");
    }
    return;
  }
  if (phase === "calibrating") {
    prevT = t;
    if (t - calT < 4.0) return;
    const drift = (nav().pitch - calErr0) / (t - calT);
    fire(POS[0], NEG[0], -CAL_N);
    if (drift < -0.005) {
      gAbs = -drift / CAL_N;
      phase = "active";
      console.log(`[podium] gain ${gAbs.toFixed(4)} deg/s per click, fps~${fps.toFixed(0)} — engaged`);
    } else {
      calErr0 = nav().pitch;
      fire(POS[0], NEG[0], CAL_N);
      calT = t;
      console.log(`[podium] calibration inconclusive (drift ${drift.toFixed(3)}), retrying`);
    }
    return;
  }

  // ---- active control ----
  const s = nav();
  const range = s.x > 0 ? Math.hypot(s.x, s.y, s.z) : num("#range .rate");

  // online attitude-gain re-estimate (err_dot = -gAbs * rr)
  if (prev) {
    for (let i = 0; i < 3; i++) {
      if (Math.abs(prev.rr[i]) >= 3) {
        const g = (prev[ERRK[i]] - s[ERRK[i]]) / (dt * prev.rr[i]);
        if (g > 0.0005 && g < 2.0) gAbs = 0.7 * gAbs + 0.3 * g;
      }
    }
  }

  // attitude: rate-knob schedule tightens as errors shrink
  for (let i = 0; i < 3; i++) {
    const err = s[ERRK[i]];
    const rrMax = Math.abs(err) > 5 ? 40 : Math.abs(err) > 2 ? 15 : Math.abs(err) > 0.5 ? 5 : 2;
    const rrTarget = clamp(Math.round(-angleRateCmd(err) / gAbs), -rrMax, rrMax);
    fire(POS[i], NEG[i], rrTarget - s.rr[i]);
  }

  const aligned = Math.abs(s.pitch) < 7 && Math.abs(s.yaw) < 7 && Math.abs(s.roll) < 10;
  if (aligned) {
    // lateral nulling: SI command -> frame units at the actuator boundary
    fire(() => translateRight(), () => translateLeft(),
      pulsesNeeded(lateralRateCmd(s.y, s.x) / fps, s.mvy, dvT()));
    fire(() => translateUp(), () => translateDown(),
      pulsesNeeded(lateralRateCmd(s.z, s.x) / fps, s.mvz, dvT()));

    // closing: corridor + docking-gate discipline, frame-unit floor/ceiling
    const corridor = Math.max(0.15, 0.10 * s.x);
    const latOK = Math.abs(s.y) < corridor && Math.abs(s.z) < corridor;
    const gatesOK = Math.abs(s.y) < 0.2 && Math.abs(s.z) < 0.2 &&
      Math.abs(s.pitch) < 0.15 && Math.abs(s.yaw) < 0.15 && Math.abs(s.roll) < 0.15 &&
      Math.abs(s.mvy) < 0.003 && Math.abs(s.mvz) < 0.003;
    const mvTarget = !latOK ? 0.0
      : (s.x < 4 && !gatesOK) ? 0.0
      : clamp(approachRateSI(range) / fps, MV_CONTACT, MV_FAR);
    fire(() => translateForward(), () => translateBackward(),
      pulsesNeeded(mvTarget, s.mvClose, dvT()));
  }
  prev = s; prevT = t;

  if (t - lastLog > 5) {
    lastLog = t;
    console.log(`[podium] t=${t.toFixed(0)}s range=${range.toFixed(1)}m ` +
      `close=${(s.mvClose * fps).toFixed(2)}m/s (${s.mvClose.toFixed(4)}/f) ` +
      `lat=(${s.y.toFixed(1)},${s.z.toFixed(1)})m ` +
      `pyr=(${s.pitch.toFixed(1)},${s.yaw.toFixed(1)},${s.roll.toFixed(1)})deg ` +
      `fps=${fps.toFixed(0)} ${aligned ? "" : "[aligning]"}`);
  }
  for (const sel of ["#success", "#fail"]) {
    const el = document.querySelector(sel);
    if (el && getComputedStyle(el).display !== "none" && +getComputedStyle(el).opacity > 0.5)
      stop("sim ended: " + sel.slice(1).toUpperCase());
  }
}, 1000);

function stop(why) {
  engaged = false; clearInterval(ctl);
  console.log(`[podium] disengaged${why ? " — " + why : ""}`);
}
window.podiumAP = { stop: () => stop("manual"), nav };
console.log("[podium] armed — waiting for the intro flight to settle");
})();

How it works

Three laws, straight from podium/control/docking.py:

1. Closing-rate profile. approach_rate(range) commands an exponential-decay glideslope in range, ramping down to a firm 0.16 m/s at contact (inside the sim's 0.2 m/s gate). The far-field ceiling is capped at 1 m/s here — the sim's thrusters deliver 5 mm/s per pulse, so speed is paid for in click count.

2. Gain-scheduled lateral/attitude nulling. A saturated proportional law with a shorter time constant near the port. The scheduling is not cosmetic: with pulsed actuation, a proportional law stalls once its command drops below half a pulse, leaving a standing offset of tau · Δv/2. With the far-field gains alone that stall offset is 0.6 m / 0.25° — outside the docking gates. The near-zone gains put it at 0.1 m / 0.05°. (Podium's click-dynamics unit tests caught exactly this failure mode.)

3. Half-pulse deadband allocator. pulses_needed() converts a rate error into an integer click count, rounding toward zero inside half a pulse — the vehicle never chatters around its setpoint.

The browser side adds the navigation and sequencing a real GNC stack would provide. The control mapping was system-identified offline with Playwright (repo: tools/ui/sysid*_issim.py): translation pulses apply in the capsule body frame in meters per frame (so the script counts requestAnimationFrame to convert its SI laws, and holds translation until the docking axis is aligned — thrusting while tumbling points the thrust vector anywhere); the displayed x/y/z map to body −z/+x/+y; velocities are read exactly from the sim's own state rather than differenced from the 0.1 m HUD; and the attitude HUD's "rate" is sign- inverted relative to the true error drift, with a frame-rate-dependent plant gain. At engage the autopilot waits for the intro camera flight to settle, then fires a 5-pulse pitch probe to measure that gain (and keeps re-estimating it in flight). This exact script, injected by the Playwright harness (tools/ui/test_issim.py), flew the sim to its official SUCCESS screen.

Caveats

Educational demo, not affiliated with SpaceX. The script reads the sim's displayed telemetry and state variables and calls the same control functions its buttons call — nothing else. If SpaceX changes the page internals, the selectors and mapping may need re-identification (the sysid scripts in the repo automate that). Paste it any time after BEGIN; it waits for the intro flight to finish on its own.
